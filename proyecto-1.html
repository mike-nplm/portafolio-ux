Sancus · Plataforma financiera B2B

Diseño de una plataforma financiera empresarial para simplificar procesos complejos como onboarding, KYB, cuentas, tarjetas, solicitudes de abono y operación administrativa.

Contexto

Sancus es una plataforma financiera B2B pensada para empresas que necesitan administrar recursos, solicitar abonos, operar cuentas y gestionar tarjetas para sus equipos o tarjetahabientes.

El reto principal era traducir una operación financiera y regulatoria compleja en una experiencia clara, entendible y accionable para usuarios administrativos.

Problema

La experiencia debía resolver tareas críticas sin generar confusión: registro de empresa, validación documental, declaración de origen de fondos, solicitud de tarjetas y operación diaria.

  • Procesos financieros difíciles de entender para usuarios no técnicos.
  • Necesidad de explicar requisitos legales sin saturar la interfaz.
  • Flujos largos con riesgo de abandono o errores de captura.
  • Coordinación constante entre negocio, producto y desarrollo.
Imagen: problema / flujo original

Proceso de diseño

Trabajé en la definición de arquitectura de información, flujos de usuario, wireframes, pantallas de alta fidelidad y documentación para desarrollo.

El enfoque fue reducir ambigüedad, ordenar la información por prioridad y separar claramente las acciones principales de las secundarias.

  • Mapeo de flujos de onboarding y operación.
  • Definición de estados, requisitos y jerarquía de información.
  • Diseño de pantallas para plataforma operativa.
  • Handoff con notas funcionales para desarrollo.
Decisión clave: priorizar acciones como solicitud de abono, solicitud de tarjetas y dispersión, evitando que el usuario tuviera que interpretar conceptos financieros complejos antes de actuar.
Imagen: arquitectura / wireframes

Solución

La solución se estructuró como una plataforma modular con flujos separados para onboarding, perfil empresarial, solicitudes, operación y consulta de estados de cuenta.

Se usó una jerarquía visual clara para que los usuarios pudieran identificar rápidamente qué información debían completar, qué acciones estaban disponibles y qué pasos seguían.

Imagen: UI final / plataforma

Impacto

Este proyecto ayudó a convertir una operación financiera compleja en una experiencia más clara, documentada y preparada para implementación.

B2B Flujos para operación empresarial
KYB Validación y documentación de empresas
UX Reducción de complejidad operativa

Aprendizajes

En productos financieros B2B, el diseño no sólo debe verse claro: también debe ayudar a ordenar decisiones de negocio, restricciones legales y necesidades operativas.

Documentar supuestos, estados y decisiones de diseño fue clave para mantener alineados a los equipos y reducir ambigüedad durante la implementación.
